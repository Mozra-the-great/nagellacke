## Reporting a Vulnerability

Wir nehmen die Sicherheit unserer Software sehr ernst. Wenn du eine Sicherheitslücke entdeckt hast, melde diese bitte **nicht** öffentlich über die GitHub Issues, sondern folge diesem Prozess:

### Wie du eine Schwachstelle meldest

Bitte sende eine detaillierte E-Mail an **261752434+Mozra-the-great@users.noreply.github.com** oder nutze – falls für dieses Repository aktiviert – die Funktion für [Private Vulnerability Reporting](https://github.com/Mozra-the-great/nagellacke/security/advisories/new) direkt auf GitHub.

Damit wir das Problem schnellstmöglich beheben können, lass uns bitte folgende Informationen zukommen:
* **Beschreibung:** Eine genaue Erklärung der Schwachstelle und der potenziellen Auswirkungen.
* **Reproduktion:** Eine Schritt-für-Schritt-Anleitung (oder ein PoC-Skript), um das Problem nachzustellen.
* **Betroffene Komponenten:** Welche Module, Funktionen oder Endpunkte sind betroffen?

### Was dich erwartet (Unser Versprechen)

* **Schnelle Rückmeldung:** Wir bestätigen den Erhalt deiner Meldung innerhalb von **48 Stunden**.
* **Transparenz:** Wir halten dich über den Fortschritt der Analyse und der Behebung auf dem Laufenden (Updates mindestens alle 7 Tage).
* **Koordination:** Wenn die Sicherheitslücke akzeptiert wird, arbeiten wir an einem Patch. Wir bitten dich im Sinne der *Responsible Disclosure*, keine Details öffentlich zu machen, bevor ein Fix bereitsteht.
* **Anerkennung:** Sobald die Lücke geschlossen ist, wirst du – wenn du das möchtest – in den Release-Notes namentlich als Entdecker genannt.
